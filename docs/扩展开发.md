## 项目结构

```
glm-snipe-extension/
├── manifest.json           # Chrome MV3 manifest
├── build.js                # 打包脚本: ES Module import 内联为单文件
├── src/
│   ├── background.js       # Service Worker（常驻后台）
│   ├── content.js          # 页面扫描、DOM 监听、点击流程（ISOLATED world）
│   ├── inject.js           # fetch/XHR 注入（MAIN world，通过 web_accessible_resources）
│   ├── popup.html/css/js   # 扩展弹窗 UI
│   ├── constants.js        # 共享常量（被内联到各 bundle）
│   ├── utils.js            # 共享工具函数（被内联到各 bundle）
│   └── notify.wav          # 提示音
├── icons/                  # 扩展图标
└── test/                   # 单元测试
```

### 构建产物 (`glmhelp/`)

`node build.js` 将 `src/` 中的 ES Module 源码打包为 MV3 兼容的单文件：

- `import { ... } from './constants.js'` 被内联替换为变量定义
- `export` 关键字被去除
- 生成 `glmhelp/` 目录，可直接作为 EasyBR 扩展管理的加载目录

## 构建与加载

```bash
# 构建
node build.js          # 或 npm run build
```

构建产物位于 `glmhelp/`。

在 EasyBR 中加载：

1. 打开 EasyBR 浏览器环境
2. 访问 `chrome://extensions/`
3. 开启右上角 **开发者模式**
4. 点击 **加载已解压的扩展程序**
5. 选择 `glmhelp/` 目录

## 连接 EasyBR 浏览器（Playwright CDP）

每个 EasyBR 浏览器环境暴露独立的 CDP 端口，通过 WebSocket 连接：

```
ws://127.0.0.1:<EASYBR_PORT>/devtools/browser/<UUID>
```

例如：

```
ws://127.0.0.1:34443/devtools/browser/8fe63165-a8bc-4f08-aff5-d3f1c5c9a44d
```

### 查找 CDP URL

**方式 1：EasyBR API**

```bash
# 获取所有环境列表及其 CDP 信息
curl -s http://127.0.0.1:<api_port>/api/env/list | jq '.[] | {id, name, cdp: .debugUrl}'
```

**方式 2：EasyBR 界面**

在 EasyBR 环境窗口的地址栏打开 `chrome://inspect`，查看 Remote Debugging 地址。

### 使用 Playwright 连接

在 Claude Code 中通过 Playwright MCP 连接：

```
skill_mcp(mcp_name="playwright", cdp_url="ws://127.0.0.1:<PORT>/devtools/browser/<UUID>", tool_name="browser_run_code_unsafe", arguments={"code": "..."})
```

常用操作模式：

```javascript
// 导航到页面
await page.goto(url, {waitUntil:'domcontentloaded',timeout:20000});
await page.waitForTimeout(3000);

// 执行 JS（注意 shadow DOM 穿透）
await page.evaluate(function() {
  // 可操作 chrome.* API、extension shadow DOM
});

// 控制台日志由 Playwright 自动捕获到 .playwright-mcp/console-*.log
```

## 扩展调试

### 查看日志

扩展通过 `console.log` 输出日志，前缀 `[GLM Snipe]` 用于区分：

| 前缀 | 来源 | 查看位置 |
|------|------|----------|
| `[GLM Snipe SW]` | background.js (Service Worker) | `chrome://extensions` → 扩展详情 → **Inspect views: Service Worker** → Console |
| `[GLM Snipe]` / `[GLM Snipe DBG]` | content.js (Content Script, ISOLATED world) | 页面 DevTools → Console |
| `[GLM Snipe Inject]` | inject.js (MAIN world) | 页面 DevTools → Console |

Playwright 连接时自动捕获页面 Console，输出到 `.playwright-mcp/console-*.log`。

### 日志前缀速查

```
[GLM Snipe SW]       → background.js 行为（注入、状态同步）
[GLM Snipe]           → content.js 关键事件（检测页面、启动扫描、点击购买）
[GLM Snipe DBG]       → content.js 心跳扫描详细日志
[GLM Snipe] CAPTCHA   → 检测到安全验证弹窗
[GLM Snipe Inject]    → inject.js fetch/XHR hook
```

### 重新加载扩展

在 EasyBR 中通过 Playwright 自动化刷新扩展：

```javascript
await page.goto('chrome://extensions/?id=<EXTENSION_ID>');
await page.evaluate(function() {
  const mgr = document.querySelector('extensions-manager');
  const sr = mgr.shadowRoot;
  const dv = sr.querySelector('extensions-detail-view');
  const dvr = dv.shadowRoot;
  const btn = dvr.querySelector('.icon-refresh');
  if (btn) btn.click();
});
await page.waitForTimeout(3000);
```

> 扩展 ID 由 EasyBR 浏览器环境决定，在 `chrome://extensions` 页面查看。

### manifest.json 变更注意事项

`content_scripts` 中新增的 URL 匹配模式在重新加载后可能**不会立即生效**——Chrome 缓存旧的匹配规则。解决办法：

1. 卸载扩展
2. 关闭 EasyBR 浏览器环境
3. 重新打开环境
4. 重新加载扩展

Service Worker 的 `scripting.executeScript` 动态注入不受此缓存影响，因此 `team-coding-detail` 页面采用 SW 动态注入而非依赖 content_scripts。

**多环境策略**：

每个 EasyBR 环境拥有独立的：
- 浏览器指纹（Canvas、WebGL、AudioContext、Fonts）

## 团队详情页支持 (team-coding-detail)

### 为什么需要动态注入

`glm-coding` 页面由 manifest.json `content_scripts` 自动注入内容脚本。但 `team-coding-detail` 页面因为 manifest 缓存问题，在 EasyBR 中无法保证 content_scripts 立即生效。

因此采用 **background.js 动态注入**，通过 `scripting.executeScript` 在 tab 加载时注入：

```javascript
// background.js
const injectedTeamDetailTabs = new Set();

chrome.tabs.onUpdated.addListener((tabId, changeInfo, tab) => {
  if (changeInfo.status === 'loading' && tab.url?.includes('team-coding-detail')) {
    injectTeamDetailContent(tabId);
  }
});

async function injectTeamDetailContent(tabId) {
  if (injectedTeamDetailTabs.has(tabId)) return;  // 去重
  await chrome.scripting.executeScript({
    target: { tabId },
    files: ['content.js']
  });
  injectedTeamDetailTabs.add(tabId);
}

chrome.tabs.onRemoved.addListener(tabId => {
  injectedTeamDetailTabs.delete(tabId);
});
```

### 页面加载时序

```
tabs.onUpdated (loading, URL 含 team-coding-detail)
  ├─ inject.js (MAIN world, 通过 content_scripts manifest, document_start)
  └─ content.js (ISOLATED world, 通过 scripting.executeScript)

content.js 内部流程:
  1. 检测 URL 包含 'team-coding-detail' → 走团队详情分支
  2. 启动 MutationObserver (监听 CAPTCHA 弹窗)
  3. 启动心跳定时器 (1200ms)
  4. 每轮心跳:
     a. checkSecurityGate() — 检测 CAPTCHA
     b. handleTeamDetailPurchase() — 下单流程
  5. 下单流程:
     a. 选择套餐（如未选中）
     b. 勾选余额支付（如存在）
     c. 勾选用户协议
     d. 点击「立即购买」
     e. 处理确认对话框
     f. 检测购买结果
```

### 日志分析要点

典型日志流：

```
[GLM Snipe] 检测到团队详情页, 启动详情页处理   ← 注入成功
[GLM Snipe] MutationObserver 已启动            ← CAPTCHA 监听就绪
[GLM Snipe] 启动团队详情页扫描                  ← 心跳开始
[GLM Snipe DBG] teamCards found: 0             ← 非个人页面卡片，预期
[GLM Snipe] 点击立即购买                        ← 下单
[GLM Snipe] 已切换余额支付                       ← 余额支付
[GLM Snipe] 已勾选用户协议                      ← 协议勾选
```

**常见诊断**：

| 日志现象 | 可能原因 |
|----------|----------|
| `teamCards found: 0` 循环无"点击立即购买" | CAPTCHA 弹窗阻止（`checkSecurityGate` 返回 true） |
| `STORAGE_KEYS has already been declared` | content_scripts 与 executeScript 同时注入导致重复。确认 manifest 中 `content_scripts` 的 `matches` 不含 `team-coding-detail*`，仅靠 executeScript 注入 |
| 只看到「检测到团队详情页」无后续 | `handleTeamDetailPurchase()` 条件不满足（检查 popup 配置中 team target 是否开启） |
| "点击立即购买"后无确认对话框日志 | 支付直接跳转新页面或 SPA 导航 |

## 常见问题

### Q: 扩展加载后没反应，日志不显示

1. 确认 URL 匹配：打开 `chrome://extensions` → 扩展详情 → **Inspect views: Service Worker**，查看 background.js 是否有 `tabs.onUpdated` 触发
2. 确认 `host_permissions` 包含 `https://bigmodel.cn/*`（manifest.json）
3. 确认 EasyBR 中扩展已开启（不是灰色）

### Q: manifest 改了但内容脚本不注入

Chrome 对 `content_scripts` 的 URL 匹配有缓存。EasyBR 中必须：
1. 卸载扩展
2. 关闭浏览器环境
3. 重新打开环境
4. 重新加载扩展

不受缓存影响的方式：在 background.js 中用 `scripting.executeScript` 动态注入。

### Q: EasyBR 中找不到 CDP 端口

通过 EasyBR API 获取：

```bash
curl http://127.0.0.1:<api_port>/api/env/list | jq '.[] | {id, name, cdp: .debugUrl}'
```

### Q: 构建产物里还有 `import` 残留

```bash
# 检查
grep -n "import " glmhelp/content.js    # 不应有结果
```

如果残留，说明 `build.js` 的正则替换没覆盖到。检查 import 语句格式是否是 `from './xxx'`（相对路径，无后缀问题）。
